# 窗口敏感性分析报告

## 问题描述

`BT_START_DATE=2025-04-10` vs `2025-04-11` 产生显著不同的回测结果：
- 0410: +143.72%, Sharpe 3.95, 551 仓位
- 0411: +125.75%, Sharpe 3.55, 570 仓位
- 差距: 17.97% 收益, 0.40 Sharpe

## 根因分析

### 1. 信号系统验证 ✅

信号生成是**纯价格驱动**的——849 个共同仓位的退出时间、退出价格、SL、TP **完全一致**（零差异）。信号逻辑没有路径依赖。

### 2. 分歧机制

384 个不同仓位的来源：

| 分类 | 数量 | 占比 | 含义 |
|------|------|------|------|
| BLOCKED_BY_UNIQUE | 256 | 66.7% | 对方持有不同仓位，`!this.position` 门禁阻断 |
| FREE (根因) | 128 | 33.3% | 对方空闲但 cooldown/状态计时不同 |
| BLOCKED_BY_COMMON | 0 | 0.0% | 共同仓位从未造成分歧 |

### 3. 级联因果链

```
种子: 0410 在 Apr 10 开了一个仓位 (0411 没有这天的数据)
  ↓ 这个仓位占用 473 bars, 在 Apr 12 08:10 平仓
  ↓ cooldown 18 bars → 0410 最早 Apr 12 ~09:40 可交易
  ↓ 0411 在 09:40 开了 LONG (0410 刚出 cooldown)
  ↓ 0410 在 09:50 开了 SHORT (0411 被占用)
  ↓ 级联: 2 根因事件 → 384 不同仓位 (3.0x 放大)
```

### 4. 本质原因

这是"一次一仓位"策略架构的**数学必然性**：

- 只能持有 1 个仓位 → 持仓期间所有信号被忽略
- 信号是稀疏的 (中位间隔 86 bars ≈ 7 小时)
- 一旦错过一个信号 → cooldown 偏移 → 后续信号窗口全部偏移
- 偏移的信号 → 不同仓位 → 不同 PnL → 不同余额 → 不同仓位大小

### 5. 尝试过的修复

| 方案 | 结果 | 原因 |
|------|------|------|
| balanceStep 阶梯化余额 | ❌ 收益 143% → 8% | 阶梯太粗, 杀死复利 |
| 窗口化亏损检测 | ❌ 改变策略行为 | 改变了 consecutiveLoss 触发条件 |
| 时钟对齐 cooldown | ❌ 改变策略行为 | 固定栅格不匹配信号分布 |
| pause 栅格对齐 | ≈ 无效果 | pause 不是实际阻断原因 |
| latent signal (持仓期间保留信号) | ❌ 仓位数翻倍 | 绕过了 cooldown 设计意图 |
| 减少 pause 时间 | ≈ 无效果 | pause 从未是有效阻断 |

**结论: 无法在不改变策略核心行为的前提下消除窗口敏感性。**

## Monte Carlo 窗口扫描结果

10 次连续天偏移扫描 (基准: 2025-04-10):

| 统计 | 值 |
|------|-----|
| 收益范围 | 116% ~ 206% |
| P25 收益 | 133.80% |
| 中位数收益 | **147.49%** |
| P75 收益 | 149.39% |
| CV (变异系数) | **15.7%** |
| Sharpe 范围 | 3.49 ~ 5.04 |
| PASS 率 | 6/10 (60%) |
| 全部盈利 | ✅ 10/10 |

## 结论

1. **窗口敏感性是"一次一仓位"策略的固有属性**, 不是 bug
2. **策略在所有窗口下都大幅盈利** (最差 +116%)
3. **中位收益 +147%** 显示策略的期望值强劲
4. **CV=15.7%** 在量化策略中属于**中等偏低的敏感性**
5. 建议部署前运行 `BT_MONTE_CARLO=10` 验证窗口稳健性
