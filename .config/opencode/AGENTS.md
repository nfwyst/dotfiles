# AGENTS.md

本文件定义当前代理的默认协作规范。

## 适用范围

- 本文件作用于当前目录及其所有子目录。
- 若子目录存在更深层的 `AGENTS.md`，则该子目录内文件优先遵循更深层规则。

## 指令优先级

- 用户当前消息 > 更深层目录的 `AGENTS.md` > 本文件 > 工具默认行为。
- 若规则冲突，遵循优先级更高者。
- 若需求不明确，先读取上下文并以最小必要改动完成任务。

## 通用规则

- 全程使用中文回复。
- 任务完成后清理本次引入的无用代码、文件和残留进程。
- **注释保护：禁止删除任何现有代码注释。例外：仅允许删除你本人在本次对话中刚刚添加且立即发现错误的注释。**
- 优先并行执行彼此独立的子任务。
- 未经用户明确要求，不要自动执行 `git commit`、`git push`、创建 PR 或其他会影响共享状态的操作。

## 工作方式

- 修改前先阅读相关文件和上下文，避免盲改。
- 优先做最小必要改动，禁止顺手重构无关代码。
- 优先复用现有实现，非必要不新增抽象。
- 不为假设场景补充额外兼容层、回退逻辑或过度防御。
- 发现阻塞时优先定位根因，避免用跳过校验、绕过钩子或破坏性操作作为捷径。

## Git 工作流

- `push` 前先执行 `git pull --rebase`。
- 禁止使用 `--force` 或 `--force-with-lease`。
- 未经用户明确确认，不执行会改写历史或难以回退的 Git 操作。

## 进程管理

- 启动进程前先检查是否已有相同进程，避免冲突。
- 仅清理与当前任务相关的残留进程；不终止来源不明或明显属于用户工作流的进程。
- 任务结束后清理本次启动的残留进程。

## 工具链

- 包管理优先级：`bun` > `npm`，`bunx` > `npx`，`uv` > `pip`。
- 安装 skill、mcp、tool 或项目依赖时同样遵循上述优先级。
- 必须通过 ESLint 和 TypeScript 类型检查。
- 禁止执行 `openclaw doctor --fix`。

## 编码风格

- 优先保持数据不可变，使用新对象/数组而非原地修改。
- 优先使用 spread、map、filter 等方式，避免 `push`、`splice`、`delete` 等原地变更。
- 文件长度尽量不超过 800 行，函数长度尽量不超过 50 行，嵌套层级尽量不超过 4 层；超出时主动拆分。
- 禁止硬编码魔法数字、URL、密钥，提取为常量或配置。
- 在系统边界（API 入口、外部数据、用户输入）执行 schema 校验（如 zod / yup），并快速失败。
- 显式处理错误，禁止静默吞异常。

## 验证要求

- 涉及 TypeScript、JavaScript 或前端行为的改动后，至少执行与改动相关的 ESLint 和 TypeScript 类型检查。
- 若仓库提供测试能力，执行与本次改动直接相关的最小测试集。
- 无法执行验证时，需要明确说明原因与未验证范围。

## 安全规范

- 禁止在代码中硬编码密钥、Token 或密码，一律使用环境变量或密钥管理服务。
- 新增依赖前检查已知漏洞，优先选择维护活跃、许可证明确的包。
- 不生成、不扩散、不提交敏感信息。

## 高风险操作

- 以下操作默认需要先获得用户确认：删除文件、批量重构、修改依赖、改动 CI/CD、数据库破坏性变更、`git push`、创建/关闭 PR、发送外部消息。
- 若操作可能影响共享状态、不可逆或超出当前任务范围，先确认再执行。

## 项目特定规则

- 字节前端项目中，新增文案须添加 `###` 前缀（Starling 识别标记）。
