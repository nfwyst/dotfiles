# OpenCode 通用指令

## 通用规则

- 语言：全程使用中文回复。
- 收尾清理：任务完成后删除所有引入的无用代码或文件。
- **注释保护：禁止删除任何现有代码注释。例外：仅允许删除你本人在本次对话中刚刚添加且立即发现错误的注释。**
- 并行优先：尽量并行执行独立子任务以提升效率。
- 任务完成后不要自动提交 git commit。

## Git 工作流

- push 前先 `git pull --rebase`，禁止使用 `--force` 或 `--force-with-lease`。

## 进程管理

- 启动进程前先检查并终止相同进程，避免冲突；任务结束后清理残留进程。

## 工具链

- 包管理优先级：bun > npm，bunx > npx，uv > pip。安装 skills/mcp/tool 时同样遵循此优先级。
- 必须通过 ESLint 和 TypeScript 类型检查。
- 禁止执行 `openclaw doctor --fix`。

## 编码风格

- 不可变性优先：创建新对象/数组，禁止原地修改，使用 spread / map / filter 代替 push / splice / delete。
- 文件上限 800 行，函数上限 50 行，嵌套上限 4 层；超出时拆分。
- 禁止硬编码魔法数字、URL、密钥，提取为常量或配置项。
- 系统边界（API 入口、外部数据、用户输入）执行 schema 校验（zod / yup），快速失败。
- 显式处理错误，禁止静默吞异常。

## 安全规范

- 代码中禁止硬编码密钥/Token/密码，一律使用环境变量或密钥管理服务。
- 新增依赖前检查已知漏洞，优先选择维护活跃、许可证明确的包。

## 项目特定规则

- 字节前端项目：新增文案须添加 `###` 前缀（Starling 识别标记）。
