# OpenCode 通用指令

个人 dotfiles 仓库，涵盖 Neovim、nushell、tmux、OpenCode 等开发环境配置。

## 通用规则

- 语言：全程使用普通话，所有回复必须为中文。
- 国际化：新增文案须添加 ### 前缀（Starling 识别标记）。
- 收尾清理：任务完成后删除所有引入的无用代码或文件。
- 注释保护：保留既有注释，仅可删除由你先前添加且现已无用的注释，严禁误删他人注释。
- 任务管理：你拥有100个不同角色的员工可供你同时使用，请尽量充分利用以提升效率，且任务完成后不要自动提交 git commit。

## 进程管理

- 命令执行或新进程启动后，立即清理相关进程，杜绝残留。
- 启动任何进程前，先检查并终止相同进程，确保无冲突运行。

## 代码校验与工具链

- 必须通过 ESLint 和 TypeScript 类型检查。
- 优先使用 bun / bunx，仅在不可用时回退至 npm / npx。

## 编码风格

- 不可变性优先：始终创建新对象/数组，禁止原地修改（mutation），使用 spread / map / filter 代替 push / splice / delete。
- 文件体积：单文件 200–400 行为宜，绝对上限 800 行；超出时必须拆分。
- 函数体积：单函数不超过 50 行，嵌套不超过 4 层；超出时提取子函数。
- 禁止硬编码：魔法数字、URL、密钥等必须提取为常量或配置项。
- 输入校验：在所有系统边界（API 入口、外部数据、用户输入）执行校验，优先使用 schema 验证（zod / yup），校验失败应快速失败并给出清晰错误信息。
- 错误处理：每层显式处理错误，禁止静默吞异常；面向用户展示友好提示，服务端记录详细日志。

## 安全规范

- 零密钥泄露：代码中禁止硬编码任何密钥/Token/密码，一律使用环境变量或密钥管理服务；发现泄露须立即轮换。
- SQL 注入防护：所有数据库查询必须使用参数化查询或 ORM，禁止字符串拼接 SQL。
- XSS 防护：用户输入渲染到 HTML 前必须转义或使用框架内建的安全绑定（React JSX / Vue template 等）。
- CSRF 防护：状态变更接口必须携带 CSRF Token 或使用 SameSite Cookie。
- 认证鉴权：每个端点必须验证身份和权限，遵循最小权限原则。
- 安全错误信息：面向用户的错误信息禁止暴露堆栈、路径、数据库结构等内部细节。
- 依赖安全：新增依赖前检查已知漏洞（bun audit / npm audit），优先选择维护活跃、许可证明确（MIT / Apache）的包。
